Instructions

The assignment is to implement the mySearchFn function. This function is called whenever the input search term changes, and is used to populate the searchResults, which then get re-rendered. The items to search through are in a small array in the SearchData variable, and they do not change.

The shape of the return value of the function is like this (say the search term is "jou"):

        [{value: "Journal Batch", matches [{start: 0, end: 3}]},
        {value: "Journal Entry", matches: [{start: 0, end: 3}]}
        ...
        ]

The return array should be sorted so that the most relevant results are first, then alphabetically. Relevancy is established and ranked by its position on this list:

  1. Exact Matches
  2. Matches at the beginning
  3. Matches at the end
  4. Partial beginning of word match ("jo en" would match "Journal Entry List")
  5. Matches in the middle

Some conflicts wouldn't be completely clear how to resolve just from that list (E.g., what would win, a beginning match of "J", or an ending match of "Batch"?), so you'll have to use your judgement to think what would be intuitive for users.

Rules

  1. Only your own code, working by yourself
  2. Utility libraries like lodash, Ramda, etc are ok, but no other libraries.
  3. Ok to use other languages supported by Codepen, like CoffeeScript

Guidelines/tips

  1. Only care about the mySearchFn function - not css, etc
  2. It may be easier to return try filtering the results without providing the matches array initially.
